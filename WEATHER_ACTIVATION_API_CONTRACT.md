> 작성: Flutter(kimi) / 대상: 백엔드 클로드 (`~/work/app/project1`, Spring Boot + PostgreSQL)
> 두 기능: ① 날씨 이벤트 푸시(게시 유도) ② 동네별/스팟 날씨 피드

## 기능 ① 날씨 이벤트 푸시 (콘텐츠 생성 유도)

### 개념
백엔드 날씨 배치가 좌표별 날씨를 갱신할 때, **유의미한 이벤트(비/눈/첫눈/노을 등)를 감지**하면
그 지역에 접속 이력이 있는 사용자에게 **"지금 우리 동네에 OO! 영상 공유?"** FCM 푸시를 보낸다.
→ 탭하면 카메라가 열리고 날씨 해시태그가 미리 채워진다 → 게시 유도.

### 이벤트 감지 규칙(백엔드, 예시)
| 이벤트 | 조건(초단기실황 카테고리) | 쿨다운 |
|---|---|---|
| 비 | PTY ∈ {1,2,5,6} (비/진눈깨비/빗방울) | 좌표당 3시간 |
| 눈 | PTY ∈ {3,7} | 좌표당 3시간 |
| 첫눈 | 그 좌표/시즌 첫 PTY=눈 | 시즌 1회 |
| 노을 | 일몰 30분 전후 & SKY≤3(맑음~구름조금) | 1일 1회 |
| 폭염/한파(선택) | T1H ≥ 33 또는 ≤ -10 | 1일 1회 |

> 스팸 방지: **좌표·이벤트별 쿨다운** 필수. 또 사용자별 "야간 금지(22~7시)" 권장.

### FCM 페이로드(백엔드 → 앱)
`data` 필드 규약 (notification + data 둘 다 전송):
```jsonc
{
  "notification": { "title": "☔ 지금 우리 동네에 비가 와요", "body": "비 오는 풍경 영상으로 공유해보세요!" },
  "data": {
    "alramCd": "20",          // ★ 20 = 날씨 이벤트(앱이 이 코드로 분기)
    "weatherEvent": "rain",   // rain|snow|firstsnow|sunset|heat|cold
    "suggestedTag": "#비 #우리동네날씨",  // 카메라/등록 화면에 미리 채울 해시태그
    "nx": "58", "ny": "127",
    "locationName": "서울 은평구"
  }
}
```
- 앱은 `alramCd=="20"`이면 탭 시 **카메라 진입 + suggestedTag 프리필**.
- 백엔드는 대상 사용자 토큰을 좌표 접속 이력(또는 관심지역)으로 선별해 전송.

### 백엔드 체크리스트
- [ ] 날씨 배치 후 이벤트 감지 + 좌표별 쿨다운 테이블(`tb_weather_event_sent`)
- [ ] 좌표→대상 사용자 토큰 매핑(접속 이력/관심지역)
- [ ] FCM 전송(위 payload), 야간 금지·사용자 알림설정(weatherPushYn) 존중
- [ ] 사용자 알림설정에 `weatherPushYn` 추가(`/cust/updateAlarmSetting` 등 기존 설정 API에)

### Flutter 측(이번 커밋에 포함)
- FCM 탭 핸들러에 `alramCd="20"` 분기 → 카메라 진입 + suggestedTag 전달
- 알림설정 화면에 "날씨 이벤트 알림" 토글(백엔드 weatherPushYn 연동)

## 기능 ② 동네별 / 스팟 날씨 피드

### 개념
- **동네별**: 이미 피드 탭에 LOCAL(관심지역)·DIST(현위치/거리) 존재 → 재활용/강화.
- **스팟별(캠핑·낚시·골프)**: 큐레이션된 야외 스팟 목록 → 스팟 선택 시 **현재 날씨 + 그 근처 커뮤니티 영상**.
  "이번 주말 OO 캠핑장 지금 어때?" 를 영상으로 확인하는 킬러 유스케이스.

### 신규 백엔드 엔드포인트
**(1) 스팟 목록** — `GET /api/spot/list?category=camping&lat=&lon=`
```jsonc
// category: camping|fishing|golf|all, 거리순
{ "code":"00", "data":[
  { "spotId": 12, "name":"가평 자라섬 캠핑장", "category":"camping",
    "lat":37.81, "lon":127.51, "nx":60, "ny":128,
    "distanceKm": 3.2, "videoCnt": 14,
    "weather": { "currentTemp":"21","sky":"1","rain":"0","weatherInfo":"맑음" } }
]}
```
**(2) 스팟별 영상 피드** — `GET /api/spot/board?spotId=12&pageNum=0&pageSize=10`
- 반환: 기존 `BoardWeatherListData` 배열(피드와 동일 구조) — 그 스팟 반경 N km 게시물.
**(3) 스팟 현재 날씨** — (1)의 weather에 포함하거나 `GET /api/spot/weather?spotId=12`.

### 스팟 데이터 출처(백엔드)
- 초기엔 **운영자 큐레이션 테이블**(`tb_spot`: 캠핑/낚시/골프 인기 장소 + 좌표) 추천.
- 좌표로 초단기실황(이미 수집중) 조인 → weather 채움.
- spotId 반경 영상은 기존 거리쿼리(lat/lon + ST_DISTANCE) 재활용.

### Flutter 측
- "스팟별 날씨" 페이지: 카테고리 탭(캠핑/낚시/골프) + 스팟 카드 리스트(이름·거리·현재날씨·영상수)
- 스팟 탭 → 그 스팟의 영상 피드(기존 VideoScreenPage 재사용) + 상단 현재 날씨 헤더
- 진입점: 홈/날씨화면 또는 하단 탭에 추가
